# sheets — 공용 스프레드시트 구조

> 코드가 이 구조(탭 이름 + 컬럼 위치)를 그대로 기대한다. **탭 이름 변경/컬럼 삽입 금지.**
> 열을 추가하고 싶으면 반드시 **기존 열 오른쪽 끝에** 추가할 것.

## 탭 목록

| 탭 이름 | 누가 쓰나 | 용도 |
|---|---|---|
| `부족원` | 사람이 관리 | 명단 · 독립(퇴사) 여부 · PIN |
| `정기공격일자` | 사람이 후보 등록, 투표는 앱 | 월별 정기모임 투표 |
| `자연재해` | 앱이 관리 | 번개 모임 |
| `벽화` | 앱이 자동 기록 | 사진/영상 인증 로그 |
| `명예의전당` | 앱이 자동 기록 | 월별 영상 출품/투표 |
| `공지` | 앱이 자동 생성/관리 | 공지사항 |
| `인증현황` | 정산 실행 시 자동 갱신(누적) | 월별 정산 결과 — 열=월 |

## 부족원 — 사람이 직접 관리 (A열만 직접, 나머지는 앱)

| 열 | 내용 | 비고 |
|---|---|---|
| A | 이름 | **필수.** 앱의 모든 기능이 이 이름 기준 |
| B~H | (자유) | 코드가 사용하지 않음 — 연락처·독립일 메모 등 자유롭게 |
| I | PIN | **앱이 자동 기록. 손대지 말 것.** 분실 시 관리자 페이지에서 초기화 |
| J | 지원여부 | `FALSE` = 정산(지원) 제외, 빈칸/`TRUE` = 지원 대상. **웹 관리자 페이지에서 설정** (직접 편집 불필요) |

## 정기공격일자 — 후보만 사람이 등록

| 열 | 내용 | 예시 |
|---|---|---|
| A | 대상월 | `2026-07` |
| B | 날짜 후보 | `7/16(수) 20:00` |
| C | 투표 마감일 | `2026-07-10` (해당 월 첫 행에만 있어도 됨) |
| D~ | 투표자 | **앱이 자동 기록** |

> 새 달 투표를 열려면: A열에 월, B열에 후보 날짜들(행마다 하나), C열에 마감일만 적으면 끝.

## 인증현황 — 열=월 누적, 행=이름(오름차순)

정산 실행할 때마다 **해당 월 열만** 다시 계산해서 덮어쓴다. 다른 달 열은 그대로 보존(누적).
활동일자·장소·Drive 링크 같은 상세 정보는 담지 않는다(그건 `벽화` 시트에 이미 있음) — 이 시트는 **월별 O/X 현황만** 빠르게 보기 위한 용도.

| A(이름) | B(2026-06) | C(2026-07) | ... |
|---|---|---|---|
| 김광훈 | O | O | |
| 이희주 | X | 정산 취소 | |
| 정민재 | 지원 제외 | 지원 제외 | |

- 셀 값: `O`(인증 완료) / `X`(미인증) / `지원 제외`(부족원 시트 J열 FALSE) / `정산 취소`(관리 탭에서 그 달만 개별 취소)
- **월 헤더(B1~)는 그 달 정산 폴더로 가는 링크** — 클릭하면 `[정산폴더]/yyyy-MM`로 이동 (`linkStatusHeaders_`). `SETTLE_FOLDER_ID` 미설정 시 링크 없이 텍스트만
- 앱이 전적으로 관리 — 수동 편집 시 다음 정산 실행에서 덮어써질 수 있음
